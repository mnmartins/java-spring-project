Notas sobre Spring Framework

Criando um projeto Spring Boot na IDE(VSCode):

  1. Abrir a paleta de comandos (ctrl+shift+p => windows/linux ou cmd+shift+p => mac) e escrever spring para abrir o "Spring Initializr: Generate a Maven Project";
  2. Selecionar a linguagem Java;
  3. Colocar no Group Id algo como br.com.java-spring-vscode-example;
  4. Colocar no Artifact Id algo como java-spring-vscode-example;
  5. Selecionar o packaging type(JAR ou WAR);
  6. Selecionar a versão do Spring Boot(mais atual e estável);
  7. Não selecionar nenhuma dependência, somente apertar Enter;
  8. Selecionar a pasta para abrigar o projeto, normalmente numa pasta "projetos" na raiz do diretório do usuário.

    9. * Caso necessário, faça uma alteração no arquivo pom.xml: mude o artifactId do org.springframework.boot de: spring-boot-starter para: spring-boot-starter-web (artefato que contém o necessário para trabalhar com REST APIs ou MVC).

  • Sobre Dependency Injection

    - É utilizada para que ococrra a Inversion of Control (IoC) e define quais classes serão instanciadas e em quais lugares serão injetadas quando houver necessidade. Basta que seja criado um ponto de injeção da classe necessária - no construtor da classe, por exemplo - e o Spring Framework irá criar uma instância.

  • Sobre o Core Container

    - Quando a aplicação é executada todas as configs pré-definidas em classes ou arquivos xml são lidas e criadas através da IoC e destruídas quando não forem mais utilizadas. Essas dependências são chamades de beans. Esses passos definem o ciclo de vida de um Container. Para que o Spring saiba quais classes ele irá gerenciar é necessário determinar e configurar (criar pontos de injeção e definir onde injetar as instâncias) esses beans, isso pode ser feito em arquivos xml ou por meio de anotações. 

                Exemplo de configuração em xml (não muito utilizada hoje em dia): 
            < beans>
                < bean class="com.example.springboot.Produto"/>
                < bean class="com.example.springboot.ProdutoController"/>
                < bean class="com.example.springboot.ProdutoService"/>
                < bean class="com.example.springboot.ProdutoRepository"/>
            < beans>
 
            Exemplo de configuração por anotação (há 4 principais tipos):
                @Component - para a classe de objeto ou entidade;
                @Service - para classe de regras de negócio;
                @Controller - para métodos GET, POST, DELETE e UPDATE;
                @Repository - métodos de transação com o BD.

    Os dois pontos de injeção mais comuns são os construtores e os setters. Ex: 

                    @Service
                public class ProdutoService {
                    private ProdutoRepository produtoRepository;

                    public ProdutoService (ProdutoRepository produtoRepository){
                        super();
                        this.produtoRepository = produtoRepository;
                    }
                    //regras de negócio
                }

                ou 

                @Service
                public class ProdutoService {
                    private ProdutoRepository produtoRepository;

                    public void setProdutoRepository (ProdutoRepository produtoRepository){
                        this.produtoRepository = produtoRepository;
                    }
                }

    O ciclo de vida de um bean depende de seu escopo, definido com a anotação @Scope. Os tipos de escopos do Spring utilizados para web são:

            * Tipo Singleton - uma única instancia para todas as solicitações da instancia;
        * Tipo Prototype - várias instancias, uma para cada solicitação;
        * Tipo Request - uma instancia criada para cada solicitação HTTP;
        * Tipo Session - terá sua instancia preservada e utilizada pelas solicitações enquanto durar a sessão.

  • Sobre Spring Boot

    - O Spring Boot é uma extensão do Spring que utiliza a base do Spring Framework para iniciar uma aplicação de forma mais simplificada e ágil e deixá-la pronta para implementação de regras de negócio. Também já traz um servidor embutido (Tomcat ou Netty). Retira a necessidade de configuração do XML Configuration ou classe @Configuration.
